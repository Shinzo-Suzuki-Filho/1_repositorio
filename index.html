Capítulo 1: Introdução ao Git

Este e-book irá guiá-lo através dos conceitos básicos e avançados do Git e GitHub.

O que é Git? É um sistema de controle de versão distribuído. Ele serve para registrar, organizar e gerenciar alterações em arquivos, principalmente código-fonte, ao longo do tempo.

Em termos simples: o Git permite que você salve versões do seu projeto, volte no tempo quando algo dá errado e trabalhe em equipe sem bagunçar o código.

Capítulo 2: Para que Serve o Git?

1. Controle de Versões

  • Guarda o histórico completo do projeto.
  • Permite voltar para versões anteriores.
  • Mostra quem alterou o quê e quando.

2. Trabalho em Equipe

  • Várias pessoas podem trabalhar no mesmo projeto.
  • Cada desenvolvedor trabalha em sua própria cópia.
  • O Git mescla as alterações de forma segura.

3. Organização do Desenvolvimento

  • Uso de branches (ramificações) para novas funcionalidades.
  • Testes e experimentos sem afetar o código principal.
  • Correções isoladas de bugs.

4. Segurança do Código

  • Evita perda de código.
  • Cada desenvolvedor tem uma cópia completa do projeto.
  • Fácil recuperação em caso de erro.

Capítulo 3: Exemplo Prático

Imagine que você está desenvolvendo um site:

  • Fez uma alteração → Git salva
  • Algo quebrou → Git restaura
  • Um colega altera outro arquivo → Git junta tudo

Capítulo 4: Git vs GitHub

Git é a ferramenta (instalada no computador).

GitHub / GitLab / GitBucket são plataformas online que hospedam repositórios Git.

No seu caso, por exemplo, quando você cria um repositório no GitHub, você está usando o Git para versionar o código e o GitHub para armazená-lo na nuvem.

Capítulo 5: Comandos Básicos do Git

No terminal Bash:

git init      # Inicia um repositório Git
git status    # Verifica o status dos arquivos
git rm arquivo  # Remove um arquivo do repositório
git mv arquivo-antigo arquivo-novo  # Renomeia um arquivo
git add .     # Adiciona arquivos para o próximo commit
git commit -m "Mensagem"  # Salva as alterações com uma mensagem
git log       # Mostra o histórico de commits
git branch    # Lista as branches (ramificações)
git checkout nome-da-branch  # Muda para outra branch
git merge nome-da-branch  # Mescla outra branch na atual
git remote add origin URL-do-repositório  # Conecta ao repositório remoto
git push -u origin main  # Envia as alterações para o repositório remoto
git pull      # Puxa as últimas alterações do repositório remoto

Capítulo 6: Resumo

Git é essencial para qualquer desenvolvedor, pois garante controle, segurança e organização no desenvolvimento de software.
